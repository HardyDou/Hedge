## 执行摘要

为 Hedge 密码管理器添加 TOTP (Time-based One-Time Password) 动态验证码功能，使用户能够在一个应用内同时管理密码和二步验证码，提升安全性和便利性。该功能将作为密码条目的可选扩展，支持扫描 QR 码和手动输入 Secret Key 两种添加方式，在详情页面和列表页面显示实时刷新的 6 位验证码，并提供一键复制功能。

**核心价值**：
- 一站式密码+验证码管理，减少应用切换
- 符合 Local-First 理念，验证码数据完全本地加密存储
- 原生 Cupertino UI 体验，与现有功能无缝集成

---

## 问题陈述

**当前痛点**：
- 用户需要在密码管理器（Hedge）和验证码应用（如 Google Authenticator、Authy）之间频繁切换
- 验证码应用通常不支持跨平台同步，或需要额外的云服务订阅
- 验证码与密码分离存储，增加了管理复杂度

**解决方案**：
在 Hedge 中集成 TOTP 功能，使每个密码条目可选地配置动态验证码。用户可以通过扫描 QR 码或手动输入 Secret Key 添加 TOTP，验证码在详情页面和列表页面实时显示，支持一键复制。

**业务影响**：
- 提升用户留存率（减少对第三方验证码应用的依赖）
- 增强产品竞争力（对标 1Password 的 TOTP 功能）
- 强化 Local-First 价值主张（验证码数据完全本地控制）

## 功能需求

### 核心功能

**功能 1: TOTP 生成与刷新**
- **描述**: 基于 Secret Key 和当前时间生成 6 位 TOTP 验证码
- **用户流程**:
  1. 用户打开配置了 TOTP 的密码详情页面
  2. 系统读取加密的 Secret Key 并解密
  3. 使用 TOTP 算法（RFC 6238）生成 6 位验证码
  4. 每秒刷新倒计时，30 秒生成新验证码
- **边缘情况**:
  - 系统时间不准确：显示警告提示"系统时间可能不准确，验证码可能无效"
  - Secret Key 格式错误：显示错误提示"无效的 Secret Key"
- **错误处理**:
  - 解密失败：提示用户重新解锁
  - 生成失败：显示"验证码生成失败"并提供重试按钮

---

**功能 2: QR 码扫描**
- **描述**: 通过相机扫描 QR 码自动提取 Secret Key
- **用户流程**:
  1. 用户在编辑页面点击"扫描 QR 码"
  2. 移动端：打开相机扫描
  3. 桌面端：打开文件选择器选择图片
  4. 系统解析 QR 码内容（格式：`otpauth://totp/...?secret=...&issuer=...`）
  5. 自动填充 Secret Key 和 Issuer
- **边缘情况**:
  - QR 码格式不正确：提示"无法识别 QR 码，请尝试手动输入"
  - 相机权限未授予：提示用户授予权限
- **错误处理**:
  - 扫描超时：提示"扫描超时，请重试"
  - 解析失败：提示"QR 码格式不正确"

---

**功能 3: 手动输入 Secret Key**
- **描述**: 用户手动输入 Secret Key 和 Issuer
- **用户流程**:
  1. 用户在编辑页面点击"手动输入"
  2. 输入 Secret Key（支持粘贴，自动去除空格和连字符）
  3. 可选输入 Issuer（发行方名称）
  4. 点击保存
- **边缘情况**:
  - Secret Key 包含非法字符：自动过滤或提示错误
  - Secret Key 长度不符合标准：提示"Secret Key 长度应为 16-32 个字符"
- **错误处理**:
  - 输入为空：提示"请输入 Secret Key"
  - 格式错误：提示"Secret Key 格式不正确（应为 Base32 编码）"

---

### 不在范围内

- **浏览器插件集成**: MVP 阶段不支持浏览器插件自动填充 TOTP
- **HOTP (基于计数器的 OTP)**: 仅支持 TOTP，不支持 HOTP
- **自定义时间步长**: 固定为 30 秒，不支持自定义
- **Steam Guard 等特殊格式**: 仅支持标准 TOTP（RFC 6238）

### 性能需求

- **TOTP 生成**: < 10ms（不阻塞 UI 线程）
- **UI 刷新**: 每秒刷新倒计时，保持 60fps
- **内存占用**: TOTP 功能增加的内存占用 < 10MB
- **加密/解密**: Secret Key 加密/解密 < 50ms

---

### 安全与隐私

- **加密算法**:
  - Secret Key 加密: AES-256-GCM（与密码相同）
  - TOTP 生成: HMAC-SHA1（RFC 6238 标准）
- **零知识架构**: Secret Key 仅在本地加密存储，不上传任何服务器
- **内存安全**: 验证码生成后立即清除内存中的明文
- **日志策略**: 严禁打印 Secret Key 或验证码明文
