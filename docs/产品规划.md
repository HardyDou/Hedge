# Hedge 产品规划

**产品定位**: Local-First 跨平台密码管理器（个人 + 企业）
**核心价值**: 隐私至上 · 原生体验 · 零成本
**当前版本**: v1.9.0
**更新日期**: 2026-03-13

## 📊 当前状态

### ✅ 已完成功能（v1.8.0）

#### 核心功能
- 密码管理（增删改查）
- AES-256-GCM 加密 + Argon2id 密钥派生
- 生物识别解锁（Face ID/Touch ID/指纹）

- 附件支持

- iCloud Drive（iOS/macOS）
- WebDAV（所有平台）
- 冲突检测与备份

#### 用户体验

- 深色模式

- 移动端字母索引栏
- 桌面端双栏布局
- Markdown 编辑器
- 实时搜索（支持拼音）

#### 桌面端特色
- 系统托盘常驻
- 快捷面板（悬停显示详情）
- 后台运行

#### 数据导入

#### CLI（v1.9.0）

- IPC 模式（生物识别） + 独立模式（主密码）
- 会话令牌加密存储
- Unix Socket 传输

## 🏗️ 统一架构设计

### 个人版 vs 企业版

```
┌─────────────────────────────────────────────────────────┐
│                    Hedge App                            │
├─────────────────────────────────────────────────────────┤
│  我的 Vault (个人)           团队 Vault (企业)          │
│  ├─ 本地存储 (默认)          ├─ 企业服务器              │
│  ├─ WebDAV 同步              │   - 自建 (开源)           │
│  └─ 可选云服务               │   - 托管 (付费)           │
│                             └─ 权限管理                  │
│                             └─ 审计日志                  │
└─────────────────────────────────────────────────────────┘
```

### 核心原则

1. **个人版保持 Local-First**：不强制服务器，本地存储 + 可选 WebDAV
2. **企业版需要服务器**：团队数据存在服务器上
3. **统一架构**：同一套代码，两个模式可切换
4. **数据隔离**：个人 Vault 与团队 Vault 完全隔离

### 用户同时有个人密码 + 企业密码时的设计

```
┌─────────────────────────────────────────────────────────┐
│  🔍 搜索... (搜索全部密码)                    [用户]    │
├─────────────────────────────────────────────────────────┤
│  [全部] [个人] [团队-A] [团队-B]                        │  ← 筛选器
├─────────────────────────────────────────────────────────┤
│                                                         │
│  🔐 GitHub                          personal            │
│  zhangsan@email.com                                   │
│                                                         │
│  🔐 Gmail                         personal              │
│  zhangsan@gmail.com                                   │
│                                                         │
│  🏢 AWS Console (团队: A公司)     team-admin           │  ← 团队密码
│  root@aws.amazon.com                                   │
│                                                         │
│  🏢 公司 VPN (团队: B公司)       team-member           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**设计要点**：

1. **统一列表 + 筛选器**
   - 默认显示全部密码
   - 可筛选：全部 / 个人 / 具体团队
   - 适合团队数量较少的场景

2. **视觉区分**
   - 个人密码：🔐 图标 + "personal" 标签
   - 团队密码：🏢 图标 + 团队名称 + 角色标签

3. **统一搜索**
   - 搜索同时查个人和团队密码
   - 搜索结果显示来源（个人/团队）

4. **权限控制**
   - 个人密码：完全控制
   - 团队密码：受角色限制（只读/编辑/管理）

5. **多团队支持**
   - 左侧/顶部可切换不同团队
   - 团队数量多时可收起为下拉菜单

### 服务器选项

- **自建**：企业提供服务器端开源代码（Docker 部署）
- **托管**：官方提供 SaaS 服务（可选付费）
