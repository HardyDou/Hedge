## 二、设计原则

### 2.1 核心原则（来自 AGENTS.md）

1. **Native first** - 严格遵循 Cupertino 规范
2. **Security is visible** - 安全 UX 让人放心
3. **Calm hierarchy** - 信息分层，主要操作清晰
4. **Whitespace is structure** - 用间距组织内容
5. **Consistency over cleverness** - 可预测的模式建立信任

### 2.2 体验原则

| 原则 | 描述 | 示例 |
|------|------|------|
| **渐进式披露** | 复杂功能分层展示 | 新手用默认配置，高级用户显示更多选项 |
| **即时反馈** | 每个操作都有视觉反馈 | 按钮点击、加载状态、成功/失败提示 |
| **容错设计** | 允许撤销，减少恐惧 | 删除前确认、可恢复误操作 |
| **上下文关联** | 在合适的时机展示功能 | 填写密码时显示生成器入口 |
| **最小干扰** | 不打断用户工作流 | 后台同步、自动锁定不突兀 |

## 三、体验标准 Checklists

### 3.1 新用户引导 Onboarding

- [ ] 首次打开有欢迎页，介绍核心价值
- [ ] 有步骤式引导（3-5 步）
- [ ] 引导创建第一个密码
- [ ] 展示核心功能入口（生成器、搜索）
- [ ] 可跳过，但之后能再次查看
- [ ] 完成后有成就感（"已就绪"）

### 3.2 核心流程体验

#### 新增密码
- [ ] 入口明显（右上角 + 按钮）
- [ ] 表单字段清晰（标题/用户名/密码/URL）
- [ ] 密码字段有生成器入口
- [ ] 保存后有成功反馈
- [ ] 可快速再次新增

#### 查看/编辑密码
- [ ] 列表页可见标题、用户名、URL 摘要
- [ ] 点击进入详情
- [ ] 密码默认隐藏，点击/悬停可查看
- [ ] 编辑入口明显
- [ ] 删除有二次确认

#### 搜索
- [ ] 搜索框始终可见（桌面顶部/移动 nav）
- [ ] 搜索响应 < 200ms
- [ ] 支持拼音搜索
- [ ] 无结果时显示空状态
- [ ] 历史搜索记录（可选）

#### 密码生成器
- [ ] 新增/编辑密码时可触发
- [ ] 实时显示密码强度
- [ ] 可配置长度、字符类型
- [ ] 一键使用当前密码
- [ ] 记住用户偏好

### 3.3 错误处理

- [ ] 网络失败有明确提示
- [ ] 验证失败有具体原因
- [ ] 提供解决方案（重试、检查网络等）
- [ ] 不暴露敏感技术细节
- [ ] 错误文案友好、不吓用户

### 3.4 加载状态

- [ ] 页面切换有加载指示
- [ ] 长时间操作有进度条
- [ ] 加载失败有重试选项
- [ ] 骨架屏（Skeleton）优于转圈

### 3.5 空状态 Empty States

- [ ] 无密码时有引导（"添加第一个密码"）
- [ ] 无搜索结果有提示
- [ ] 有温暖的插画或图标
- [ ] 有明确的操作入口

### 3.6 平台特定体验

#### iOS
- [ ] 支持 Dynamic Type
- [ ] 支持深色/浅色/系统切换
- [ ] 列表有索引栏
- [ ] 支持 Haptic Feedback

#### macOS
- [ ] 菜单栏快捷面板流畅
- [ ] 支持快捷键
- [ ] 触摸栏支持（如适用）
- [ ] 窗口记忆

#### Android
- [ ] Material You 动态颜色
- [ ] 导航手势支持
- [ ] 指纹/面部识别

#### Linux/Windows
- [ ] 窗口管理正常
- [ ] 系统托盘功能完整
- [ ] 自动启动配置

## 四、验证机制

### 4.1 发布前检查

```
□ 功能测试通过
□ 视觉走查（对照设计稿）
□ 体验 checklist 逐项确认
□ 性能测试（启动 < 2s，搜索 < 200ms）
□ 无障碍检查（VoiceOver/TalkBack）
□ 极端场景测试（大量密码、网络离线）
```

### 4.2 用户反馈收集

| 渠道 | 用途 |
|------|------|
| App 内反馈入口 | 收集问题和建议 |
| App Store/Play Store 评论 | 监控评分和评论 |
| 社区/Discord | 用户讨论 |
| 定期用户访谈 | 深入了解痛点 |

### 4.3 指标监控

| 指标 | 目标 | 监控方式 |
|------|------|----------|
| 首次使用完成率 | > 70% | 埋点 |
| 核心功能使用率 | > 50% | 埋点 |
| 平均会话时长 | > 2min | 埋点 |
| 7 日留存 | > 40% | 埋点 |
| NPS | > 40 | 问卷 |
| Crash 率 | < 0.1% | 监控 |

## 五、持续优化

### 5.1 迭代节奏

1. **每周**：Bug 修复、小优化
